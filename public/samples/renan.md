## STAR Stories

### Story 1 — Shipping MINE from zero
- **Situation:** Agility had a green-field fashion AI product with no frontend, vague designs, and a tight window to show real users.
- **Task:** Own the frontend end-to-end — architecture, UI, API contracts — and get a credible v1 in front of users.
- **Action:** Chose Next.js + TypeScript for velocity and SSR/SEO; set up a component library so the design team could iterate in parallel; wrote the API integration layer against a still-moving backend by aligning contracts weekly; pushed for perf budgets from day one (image optimization, route-level code splitting).
- **Result:** Shipped v1 on schedule. Platform hit its early-user targets and became the flagship case study for Agility. The component library I set up was reused in later client projects.

### Story 2 — Turning a messy codebase into a shippable product
- **Situation:** Inherited a client project with inconsistent patterns, no tests, slow builds, and a release date that couldn't move.
- **Task:** Stabilize the codebase while continuing to ship features.
- **Action:** Instead of a big-bang rewrite, I introduced conventions incrementally: strict TS on new files, a shared UI primitives folder, and Playwright smoke tests around the critical paths. Paired with the team on each PR so the patterns stuck.
- **Result:** Release shipped on time with zero P0s in the first week. Build time dropped meaningfully and the team kept the conventions after I rolled off.

### Story 3 — Saying no to the wrong abstraction
- **Situation:** A teammate proposed a heavy "universal form framework" to handle every form in the app.
- **Task:** Decide whether to adopt it before we had more than three forms.
- **Action:** Pushed back: three forms isn't a pattern yet. Proposed we build each form directly and extract shared pieces only when the duplication became real. Documented the reasoning in an ADR.
- **Result:** Avoided weeks of abstraction work. When we did extract shared logic later, it was grounded in actual usage and much smaller than the original proposal.
